Executive Summary

Qubitcoin is a Rust-based Bitcoin implementation architected as a modular, embeddable system. Unlike Bitcoin Core (C++), qubitcoin is built from the ground up in Rust with several key innovations:

  • Embeddable consensus library - Pure, zero-I/O consensus validation
  • no_std support - Runs in embedded/WASM environments
  • rust-bitcoin compatibility - Bidirectional type conversion
  • Metashrew compatibility - Bitcoin metaprotocol indexer integration
  • Modular architecture - 15 separate crates with focused responsibility

This is not a fork of Bitcoin Core or rust-bitcoin, but a ground-up reimplementation of Bitcoin consensus rules in idiomatic Rust.

Architecture Overview

Workspace Structure

Qubitcoin uses Cargo workspace with 15 crates organized into 4 layers:

Qubitcoin Architecture Layers

Application Layer

  • qubitcoind - Full node daemon (equivalent to bitcoind)
  • qubitcoin-cli - Command-line interface
  • qubitcoin-tx - Transaction builder tool

Node Layer

  • qubitcoin-node - Validation, ChainstateManager, mempool
  • qubitcoin-wallet - HD wallet implementation
  • qubitcoin-rpc - JSON-RPC server
  • qubitcoin-net - P2P networking

Consensus Layer

  • qubitcoin-consensus - THE CORE - Embeddable, no_std consensus

Primitives Layer

  • qubitcoin-crypto - SHA256d, RIPEMD160, secp256k1
  • qubitcoin-primitives - Uint256, Amount, Txid, BlockHash
  • qubitcoin-serialize - Bitcoin wire format
  • qubitcoin-script - Script types and opcodes
  • qubitcoin-storage - RocksDB storage layer
  • qubitcoin-util - Utility functions
  • qubitcoin-common - Shared types

Detailed Crate Breakdown

qubitcoin-crypto

Cryptographic primitives wrapping Bitcoin's cryptographic building blocks:

  • SHA256d - Double SHA-256 for block/transaction hashing
  • RIPEMD160 - Address generation
  • secp256k1 - ECDSA signatures
  • SipHash - Hash table algorithm
Key Feature: This is the only crate that directly depends on cryptographic libraries, ensuring single point of crypto dependency management.

qubitcoin-script

Bitcoin Script implementation - the stack-based scripting language for transaction validation:

OP_DUP OP_HASH160 <pubKeyHash> OP_EQUALVERIFY OP_CHECKSIG
  • Script - Bytecode parsing/serialization
  • Opcode - All Bitcoin opcodes (OP_0 through OP_INVALIDOPCODE)
  • ScriptNum - Variable-length integer encoding
  • ScriptVerifyFlags - Soft fork activation flags (SegWit, Taproot)

The Consensus Library (The Core)

qubitcoin-consensus is the heart of the project - an embeddable consensus library with no I/O dependencies that can run in WASM and embedded environments.

Key Features

  • no_std support - Runs without Rust standard library
  • Zero I/O - Pure functions, no file/network operations
  • Embeddable - Can be linked into any application

Key Design Decisions

1. Embeddable Consensus Library

#![cfg_attr(not(feature = "std"), no_std)]

Use Cases:

  • Browser-based Bitcoin block explorer (WASM)
  • Hardware wallet firmware
  • Metaprotocol indexers (Ordinals, Runes)
  • Academic research on consensus changes

2. rust-bitcoin Compatibility

// qubitcoin -> rust-bitcoin
let btc_tx = bitcoin::Transaction::try_from(&qb_tx)?;

// rust-bitcoin -> qubitcoin
let qb_tx = Transaction::try_from(&btc_tx)?;

Uses byte-level round-trip through serialization for perfect fidelity.

Security Considerations

Memory Safety

Rust provides automatic protection against:

  • Buffer overflows
  • Use-after-free
  • Null pointer dereferences
  • Data races
vs. Bitcoin Core (C++): Manual memory management creates potential for memory bugs that are harder to audit.

Consensus Correctness

Critical: Consensus bugs can fork the blockchain! Qubitcoin mitigates this through rust-bitcoin compatibility testing, round-trip serialization verification, and comprehensive test suites.

Supply Chain Security

Dependencies are minimal and well-audited:

  • secp256k1 - Critical, audited (same as Bitcoin Core)
  • bitcoin_hashes - Audited implementations
  • tokio - Industry standard async runtime
  • rocksdb - Facebook-maintained, production-proven

Conclusion

Qubitcoin is a serious, well-architected Rust implementation of Bitcoin.

Strengths

  • Modular architecture - 15 focused crates
  • Embeddable consensus - no_std, pure functions
  • rust-bitcoin compatibility - Byte-perfect conversion
  • Memory safety - Rust prevents entire classes of bugs
  • Performance focus - Parallel validation, optimized builds
  • Modern tooling - Cargo, tracing, prometheus

Weaknesses

  • Early stage - Version 0.1.0, lacks battle-testing
  • Unknown consensus compatibility - Needs Bitcoin Core test vector validation
  • Limited documentation
  • Unproven in production

Primary Use Case

The metashrew-compat feature suggests qubitcoin is designed for metaprotocol indexing:

  • Metashrew handles indexing (Ordinals, Runes, BRC-20)
  • Qubitcoin provides consensus validation
  • Together they process Bitcoin blocks for metaprotocol data
Recommendation: Use qubitcoin-consensus for embedding Bitcoin validation in Rust projects. Avoid using qubitcoind as primary node until more mature.
